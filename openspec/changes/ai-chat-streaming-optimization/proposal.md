# AI 对话流式展示优化 · 变更提案

## 变更 ID
`ai-chat-streaming-optimization`

## 目标
优化「老朋友」对话框的流式内容展示体验，减少首字延迟与卡顿感，确保在流式响应返回时尽快展示内容。

## 背景
- 当前前端 SSE 解析依赖换行符，遇到分片或不完整行会导致内容滞留，出现明显停顿。
- React 18 自动批处理在高频状态更新下会合并渲染，导致流式输出不够顺滑。
- 流式 JSON 被拆分时解析失败，会丢失已返回的部分内容。

## 范围

### 本次实现
1. **缓冲区优化**：对缓冲区进行增量解析，尽早展示可解析内容，不必等待完整换行。
2. **禁用自动批处理**：在流式更新路径中显式同步刷新，确保每次 chunk 到达即渲染。
3. **容错解析**：对不完整 JSON 进行降级解析，尽可能提取 `content` 字段并展示已返回内容。

### 不在此次范围
- 后端 SSE 输出格式调整
- UI 样式与交互改版
- 对话历史持久化或存储层改造
