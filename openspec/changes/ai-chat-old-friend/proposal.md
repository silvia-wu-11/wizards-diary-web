# 基于日记数据的 AI 对话（老朋友）· 变更提案

## 变更 ID
`ai-chat-old-friend`

## 目标
新增「老朋友」AI 对话功能，让用户与一个拟人化的 AI 角色对话，AI 基于当前筛选条件下的日记数据作为上下文，提供回答建议或引导；若用户提及前置上下文之外的历史记忆，支持动态从日记库拉取相关数据补充上下文。

## 背景
- **产品定位**：魔法氛围日记本，希望营造「与老朋友聊天」的沉浸感，而非冷冰冰的 AI 助手
- **技术栈**：项目已有 Vercel AI SDK 能力规划，本次采用 HuggingFace API 调用云端大模型（PsyLLM-8B 或兼容模型）
- **数据基础**：首页已有 `getEntriesPaginated` 支持日期范围、tag、日记本、关键词筛选；日记本内页有 `bookEntries` 及搜索筛选

## 范围

### 本次实现

**1. 对话入口「老朋友」按钮**
- 首页：筛选搜索栏右侧，拟人形象按钮，点击展开对话框
- 日记本内页：书本闭合时，footer 中 Search 按钮左侧，非拟人形象，复用 ActionButton；书本打开时不展示

**2. AI 对话框**
- 上下文：将当前筛选条件（日期范围、日记本、tags、搜索词）+ 筛选出的日记数据作为「前置上下文」
- 初始消息：由「老朋友」给出引导性问题或打招呼
- 输入：底部输入框，用户输入问题发送，云端大模型结合日记内容回答
- 动态补充：若用户提及前置上下文之外的记忆，从日记库拉取相关日记补充到上下文后再调用大模型

**3. 云端大模型**
- 使用火山方舟（Volcengine Ark）API 调用
- 流式输出：`stream: true`，前端对话框逐字展示 AI 回复
- 环境变量：`HUOSHAN_MODEL_API_KEY`、`HUOSHAN_MODEL_ID`

### 不在此次范围
- 对话历史持久化（刷新后清空）
- 流式输出（可后续迭代）
- 多轮对话的上下文窗口管理优化（先实现基础版）
