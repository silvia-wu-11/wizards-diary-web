# 日记列表分页与搜索 · 变更提案

## 变更 ID
`diary-list-pagination-search`

## 目标
为首页日记列表和日记本内页增加分页、日期范围筛选、组合筛选及关键词搜索能力，提升大量日记场景下的浏览与检索体验。

## 背景
当前实现中：
- **首页**：一次性加载全部日记，使用单日期、tag、日记本、关键词进行前端内存筛选；无分页，数据量大时性能与体验受限
- **日记本内页**（DiaryView）：已有 Search 按钮但未实现搜索功能
- **日期选择器**：首页与日记本内页均只能选择单个日期

## 范围

### 本次实现

**首页日记列表**
1. **分页无限滚动**：日记列表采用无限滚动或虚拟列表加载，不展示分页按钮，保持瀑布流布局
2. **日期范围选择**：日期选择器从单日期改为日期范围筛选（仅首页）
3. **组合筛选**：支持日期范围、tag（精确匹配）、日记本、关键词（模糊匹配）的组合筛选

**日记本内页**
1. **关键词模糊搜索**：实现 Search 按钮触发的关键词搜索，在当前日记本内按 title/content 模糊匹配

### 不在此次范围
- 日记本内页的日期范围选择（保持单日期选择）
- 全文搜索（如 PostgreSQL full-text search）
- 搜索历史、热门搜索等增强功能

## 依赖
- 行为规格：`openspec/specs/diary-list/spec.md`（新建）
- 技术设计：见本 change 的 `design.md`
- 前置：`diary-backend-persistence` 已完成（数据在后端，可扩展 API 支持分页与筛选）
