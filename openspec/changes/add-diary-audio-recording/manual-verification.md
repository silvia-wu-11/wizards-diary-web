## 手动验收清单

适用 change：`add-diary-audio-recording`

目标：覆盖 `5.2` 的前端关键交互，以及 `5.3` 的首页 / 详情页原有能力回归验证。

## 0. 验收准备

- [x] 使用 Chromium 与 Safari 至少各验证 1 次录音流程。
- [x] 使用已登录账号进入首页，并准备一个已有日记本与一篇可编辑日记。
- [x] 首次进入页面时允许麦克风权限。
- [x] 确认对象存储与数据库已连接，保存后可刷新页面重新读取数据。

## 1. 首页录音流程

- [x] 在首页输入区确认布局顺序为“正文 -> 图片列表 -> 录音内容区”。
- [x] 确认录音按钮位于“添加图片”按钮左侧。
- [x] 点击“开始录音”后进入录音态，出现动态声波与录音中提示。
- [x] 录音过程中点击保存，页面提示“请先停止录音再保存”，且不会提交日记。
- [x] 录音满 `120s` 后自动停止，并出现最长时长提示。
- [x] 手动停止录音后，展示音频播放器、时长、展示名称输入框与“删除录音”按钮。
- [x] 修改展示名称后，预览卡片内名称同步更新。
- [x] 点击“删除录音”后弹出确认弹窗；点击“取消”时录音与名称保持不变。
- [x] 再次点击“删除录音”并确认后，录音区恢复为空态提示。
- [x] 重新录一段音频并保存日记，保存成功后首页表单被清空。
- [x] 刷新页面或重新进入对应日记后，可看到保存后的录音名称、播放器、时长信息。

## 2. 详情页新建 / 编辑录音流程

- [x] 进入 `/diary/new` 创建态，确认录音区位于左页图片区下方。
- [x] 创建态录音、删除确认、展示名称编辑行为与首页一致；已录音态仅保留“删除录音”按钮，不提供“重录”按钮（需先删除后才能重新录音）。
- [x] 新建态录音保存成功后，跳转到新日记详情页并保留录音信息。
- [x] 打开已有日记进入编辑态，能看到已保存的录音名称与播放器。
- [x] 编辑态修改展示名称后保存，刷新页面后名称仍为最新值。
- [x] 编辑态删除录音并保存，刷新页面后录音信息已被清空。
- [x] 只读态不展示录音录制控件，仅展示录音名称与播放器。

## 3. 危险确认弹窗回归

- [x] 删除录音确认弹窗的遮罩、动画、按钮布局与删除日记本弹窗风格一致。
- [x] 删除录音弹窗文案为录音删除语义，不与删除日记本文案混淆。
- [x] 删除日记本弹窗仍可正常打开、取消与确认删除。

## 4. 原有能力回归

- [x] 首页纯文本日记可正常保存。
- [x] 首页图片上传、预览、删除与保存能力未受影响。
- [x] 详情页文本编辑、标签编辑、图片上传 / 删除与保存能力未受影响。
- [x] `ViewEntryModal` 仍可正常展示正文、图片，并新增展示录音播放器。
- [x] 日记本删除流程仍可正常执行，删除后正确返回首页。
- [x] 搜索、分页滚动、切换日记本等既有交互未出现明显报错或布局破坏。
